# AR Floor Plan Demo

A no-install web AR demo. Users scan a QR code → web page opens → camera tracks a printed floor plan → 3D building model overlays on it.

## What's in this package

- **`index.html`** — the AR demo itself (the page your QR code points to)
- **`compiler.html`** — one-time setup tool to generate the target tracking file
- **`targets.mind`** — generated by `compiler.html` (you'll create this once)
- **`floorplan.png`** — the printable floor plan image (also generated by `compiler.html`)

## Setup (one time, ~5 minutes)

### 1. Generate the target file

Open `compiler.html` in any browser (works locally — just double-click it). Click through the 4 steps:
1. Render Floor Plan → 2. Compile Target → 3. Download `targets.mind` → 4. Download `floorplan.png`

Place the downloaded `targets.mind` in the same folder as `index.html`.

> **Why this step?** MindAR uses a pre-compiled binary file with extracted feature points for fast tracking. This compilation can't happen at runtime in the user's browser — it has to be done once ahead of time.

### 2. Deploy to free static hosting

The page needs HTTPS for camera access. Pick any of these (all free):

**Netlify Drop (easiest):**
1. Go to https://app.netlify.com/drop
2. Drag the folder containing `index.html` and `targets.mind` onto the page
3. Done — you get a URL like `https://random-name.netlify.app`

**GitHub Pages:**
1. Push the files to a public repo
2. Settings → Pages → enable on main branch
3. URL: `https://username.github.io/repo-name`

**Cloudflare Pages, Vercel, Surge.sh** — all similar drag-and-drop options.

### 3. Generate the QR code

Take your deployed URL and feed it into any QR generator (e.g., qr-code-generator.com). Print or display the QR code.

## How it works for users

1. User scans QR → page opens on their phone
2. Tap "Launch AR Experience" → grants camera permission
3. Aim phone at the printed/displayed floor plan
4. The 3D building model rises out of the floor plan, anchored to it

The user can move around the floor plan and view the building from different angles.

## Replacing the placeholders with your real assets

### Real floor plan
Replace the SVG inside `index.html` (the `<svg id="target-preview">` block) with an `<img src="your-floorplan.png">` tag, and replace the SVG in `compiler.html` similarly. Then re-run the compiler to regenerate `targets.mind`.

**Tracking quality tip:** floor plans with lots of varied detail (annotations, dimensions, room labels, irregular walls) track better than sparse line drawings. If yours is too clean, add a unique border, logo, or text block.

### Real 3D model
The placeholder building is procedurally generated in `index.html` inside the `createBuilding()` function. To use a real `.glb`/`.gltf` model:

1. Place your `building.glb` in the folder
2. In `index.html`, add this import:
   ```js
   import { GLTFLoader } from 'three/addons/loaders/GLTFLoader.js';
   ```
3. Replace `createBuilding()` with:
   ```js
   async function createBuilding() {
     const loader = new GLTFLoader();
     const gltf = await loader.loadAsync('./building.glb');
     const group = gltf.scene;
     // Scale & position to fit the floor plan
     group.scale.set(0.5, 0.5, 0.5);
     group.userData.targetY = 0;
     group.userData.currentY = -0.3;
     group.position.y = -0.3;
     // Add lights
     const sun = new THREE.DirectionalLight(0xfff4e0, 1.0);
     sun.position.set(2, 4, 3);
     group.add(sun);
     group.add(new THREE.AmbientLight(0xffffff, 0.55));
     return group;
   }
   ```
4. The `startAR()` function already awaits `createBuilding()` if you make it async — adjust accordingly.

## Compatibility

- iOS Safari 13+ ✓
- Android Chrome 80+ ✓
- Desktop Chrome/Firefox/Safari ✓ (with webcam)
- Requires HTTPS (or `localhost` for testing)

## Computational footprint

Everything is client-side. Zero server compute. The static files total under 200 KB plus the `.mind` file (~100–500 KB depending on image complexity) plus your 3D model. CDN-hosted libraries (MindAR, Three.js) are cached after first load.

## Local testing

Camera access requires HTTPS or `localhost`. Quickest local server:

```bash
# Python
python -m http.server 8000

# Node
npx serve

# Then open http://localhost:8000
```
